Windows使用回车和换行两个字符来结束一行，而Mac和Linux只使用换行一个字符
提交代码时自动地把行结束符CRLF转换成LF，而在检出代码时把LF转换成CRLF
$ git config --global core.autocrlf true

我的建议：在Mac上设置 autocrlf = input, 在Windows上设置autocrlf = true（默认值）。

----------------------------------------------------------------------------------------------------------------------------------

这样的话，

Windows：（true）

提交时，将CRLF 转成 LF再提交；

切出时，自动将LF 转为 CRLF;


MAC/Linux: (input)

提交时,   将CRLF 转成 LF再提交；

切出时，保持LF即可

这样即可保证仓库中永远都是LF. 而且在Windows工作空间都是CRLF, 在Mac/Linux工作空间都是LF.
